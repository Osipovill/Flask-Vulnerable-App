Сканирование приложения с помощью PT Application Inspector

Плагин PT Application Inspector помогает находить уязвимости и недокументированные возможности в исходном коде приложений. Помимо анализа кода, встроенные модули находят ошибки в конфигурационных файлах и уязвимости в сторонних компонентах и библиотеках, используемых при разработке. Плагин поддерживает следующие языки программирования: C#, Go, Java, JavaScript, Kotlin, PHP, Python, Ruby, SQL и TypeScript.

Установка и включение плагина

Для начала работы необходимо включить плагин в папке открытого проекта. При повторном открытии проекта плагин включается автоматически, сохраняя историю сканирования и действий. Плагин создаёт папку .ai в проекте, в которой хранятся база данных, файлы логов и файл конфигурации.

Установка анализатора кода

Для корректной работы плагина необходимо установить анализатор кода. Его можно загрузить автоматически, нажав Download Analyzer в интерфейсе Visual Studio Code, или вручную, скачав с официального сайта.

После загрузки установите анализатор.

Как запустить сканирование

Запустить сканирование проекта можно следующими способами:

  • Нажав на кнопку в разделе CODE SCANNING;
  • При сохранении изменений в проекте, если выбрана опция On saving;
  • Командой PT Application Inspector: Start Scan в командной палитре;
  • Командой PT Application Inspector: Start Full Scan для полного сканирования.

Процесс сканирования отображается на вкладке OUTPUT. Первое сканирование занимает больше времени, так как происходит первоначальная загрузка базы данных уязвимых компонентов.

Анализ результатов сканирования

Все найденные уязвимости отображаются на вкладке PROBLEMS и в разделе CODE SCANNING. Щелкнув по уязвимости, можно перейти к соответствующей строке кода в редакторе. Диаграмма DATA FLOW отображает, как данные проходят через приложение, и содержит следующие элементы:

  • Entry point – начальная точка контроля потока данных;
  • Data entry point – файл и строка кода, где данные поступают в приложение;
  • Data changes – описание функций, изменяющих потенциально опасные данные;
  • Exit point – строка выполнения потенциально уязвимой функции;
  • Best place to fix – строка кода, где лучше всего исправить уязвимость.

Управление уязвимостями

Плагин позволяет подтверждать или отклонять уязвимости, фильтровать их по степени опасности, статусу и другим параметрам, а также сравнивать результаты двух сканирований.
